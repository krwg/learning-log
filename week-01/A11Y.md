# Accessibility (a11y) Audit Report

## 5 Issues Found and Fixes

### 1. Missing Skip-link
- **Issue:** Keyboard users were forced to tab through the entire navigation menu on every page before reaching the main content.
- **Fix:** Added a `<a href="#main-content">Skip to main content</a>` link as the very first focusable element inside `<body>` on every page.

### 2. Missing Target Anchor for Main Content
- **Issue:** The skip-link had nowhere to navigate to — no element with a matching `id` existed on the page.
- **Fix:** Added `id="main-content"` to the `<main>` tag on all pages, so the skip-link lands exactly at the start of the primary content.

### 3. Insufficient Form Semantics
- **Issue:** Form fields lacked explicit labels or logical grouping, making them hard to understand for screen reader users.
- **Fix:** Every `<input>`, `<select>`, and `<textarea>` now has an explicit `<label for="id">` linked by `id`. Related radio buttons are wrapped in `<fieldset>` with a `<legend>` for contextual grouping.

### 4. No Active Page Indicator in Navigation
- **Issue:** Screen reader users had no way to know which page they were currently on when hearing the navigation menu.
- **Fix:** Added `aria-current="page"` to the link representing the current page in `<nav>` on each page.

### 5. Unclear Validation Requirements
- **Issue:** Users did not understand why a field with a `pattern` attribute was rejected — no explanation was provided.
- **Fix:** Added descriptive `title` attributes to fields using `pattern`, giving a human-readable explanation of the rule (e.g., "3-20 characters: letters, numbers, or underscore").

## Notes on Focus Visibility
Visible focus indicators are provided by the **browser's native behavior**. Since no custom styles override the default `outline`, the browser automatically renders a visible focus ring when navigating with the `Tab` key, satisfying the requirement without any CSS.

## Result
- **Lighthouse Accessibility score:** 95–100
- Full keyboard navigation works via `Tab` / `Shift+Tab` / `Enter` / `Space`.
- All forms are semantically correct and screen-reader friendly.
